# 可视化剧情编辑器 - 产品需求文档 (PRD)

## 一、产品概述

### 1.1 产品定位
为非程序员游戏开发者提供的可视化剧情编辑工具，支持：
- 创建多章节剧情
- 可视化编辑对话流程
- 实时预览游戏效果
- 导出与游戏兼容的数据格式

### 1.2 目标用户
- 独立游戏开发者
- 剧情策划人员
- 视觉小说创作者
- 无编程经验的内容创作者

---

## 二、核心概念

### 2.1 数据层级结构

```
游戏
└── 章节 (Chapter)
    └── 场景/节点 (Scene/Node)
        ├── 对话节点
        ├── 选项节点
        ├── 条件节点
        └── 结局节点
```

### 2.2 概念定义

| 概念 | 说明 | 示例 |
|------|------|------|
| **章节** | 剧情的独立单元，每个章节有独立的节点图 | 序章、第一章、第二章 |
| **节点** | 剧情的一个步骤，包含对话、选项等 | 小雪的自我介绍 |
| **连接** | 节点之间的跳转关系 | 对话A → 对话B |
| **起始节点** | 每个章节的第一个节点，玩家进入章节时触发 | scene_1 |

## 三、功能模块

### 3.1 章节管理模块

#### 功能列表
| 功能 | 描述 | 优先级 |
|------|------|--------|
| 创建章节 | 新建一个空白章节 | P0 |
| 删除章节 | 删除章节及其所有节点 | P0 |
| 重命名章节 | 修改章节名称 | P0 |
| 切换章节 | 在不同章节之间切换编辑 | P0 |
| 章节排序 | 调整章节顺序 | P1 |
| 章节复制 | 复制整个章节 | P2 |

#### 交互逻辑
1. **切换章节时**：
   - 清空当前画布
   - 加载目标章节的节点
   - 重置选中状态
   - 更新连接线

2. **删除章节时**：
   - 弹出确认提示
   - 删除章节及关联的所有节点
   - 自动切换到相邻章节

### 3.2 节点编辑模块

#### 节点类型

| 类型 | 图标颜色 | 功能 | 数据字段 |
|------|----------|------|----------|
| 对话 | 🟢 绿色 | 显示角色对话 | speaker, text, background, character, goto |
| 选项 | 🟡 黄色 | 玩家选择分支 | options[] (text, affection, goto) |
| 条件 | 🟣 紫色 | 条件判断跳转 | check, true_goto, false_goto |
| 结局 | 🔴 红色 | 触发结局 | ending (good/normal/bad) |

#### 节点操作

| 操作 | 描述 | 交互方式 |
|------|------|----------|
| 创建节点 | 添加新节点到画布 | 点击顶部按钮 |
| 选中节点 | 选中进行编辑 | 点击节点 |
| 移动节点 | 调整节点位置 | 拖拽节点 |
| 删除节点 | 删除节点及连接 | 属性面板删除按钮 |
| 复制节点 | 复制节点数据 | 右键菜单 (P1) |

### 3.3 连接管理模块

#### 连接规则

| 源节点类型 | 可连接目标 | 连接点数量 |
|------------|------------|------------|
| 对话 | 任意节点 | 1个 (goto) |
| 选项 | 任意节点 | 每个选项1个 |
| 条件 | 任意节点 | 2个 (true/false) |
| 结局 | 无 | 0个 |

#### 连接操作

| 操作 | 描述 | 交互方式 |
|------|------|----------|
| 创建连接 | 建立节点跳转关系 | 下拉选择 / 点击连接 |
| 修改连接 | 更改跳转目标 | 下拉选择新目标 |
| 删除连接 | 移除跳转关系 | 选择"无" |

### 3.4 属性编辑模块

#### 对话节点属性

### 3.5 数据管理模块

#### 保存机制
- 自动保存：每次操作后自动保存到 localStorage
- 手动导出：点击"导出"下载 YAML 文件

#### 数据结构

```yaml
chapters:
  start:                          # 章节 ID
    name: 序章 - 相遇             # 章节名称
    next: chapter1                # 下一章节 ID
    start_node: scene_1           # 起始节点 ID
    scenes:                       # 节点列表
      - id: scene_1
        type: dialogue
        ...
  
  chapter1:
    name: 第一章 - 了解
    next: chapter2
    start_node: cafe_scene
    scenes:
      - id: cafe_scene
        ...
```

## 四、用户操作流程

### 4.1 创建新剧情

```
1. 点击"添加章节" → 输入章节名称
2. 系统自动创建起始节点
3. 编辑起始节点内容
4. 添加后续节点
5. 连接节点
6. 继续添加直到完成
```

### 4.2 编辑现有剧情

```
1. 在章节列表选择章节
2. 画布加载该章节的所有节点
3. 点击节点进行编辑
4. 修改属性、连接等
5. 自动保存
```

### 4.3 预览剧情

```
1. 点击"预览"按钮
2. 系统生成 YAML 数据
3. 跳转到游戏场景
4. 从当前章节的起始节点开始
```

## 六、技术实现要点

### 6.1 数据隔离

每个章节的数据完全独立：
- 节点列表按章节 ID 存储
- 切换章节时清空画布，重新渲染
- 连接只在同章节节点之间有效

### 6.2 数据存储结构

```javascript
{
  chapters: [
    { id: 'start', name: '序章', startNode: 'scene_1' },
    { id: 'chapter1', name: '第一章', startNode: 'cafe_1' }
  ],
  currentChapterId: 'start',
  nodesByChapter: {
    'start': [
      { id: 'scene_1', type: 'dialogue', ... },
      { id: 'choice_1', type: 'choice', ... }
    ],
    'chapter1': [
      { id: 'cafe_1', type: 'dialogue', ... }
    ]
  }
}
```

### 6.3 切换章节流程

```javascript
function switchChapter(chapterId) {
  // 1. 保存当前章节数据
  saveCurrentChapter();
  
  // 2. 清空画布
  clearCanvas();
  
  // 3. 更新当前章节 ID
  state.currentChapterId = chapterId;
  
  // 4. 加载新章节节点
  const nodes = state.nodesByChapter[chapterId] || [];
  
  // 5. 渲染节点
  nodes.forEach(node => renderNode(node));
  
  // 6. 渲染连接线
  updateConnections();
  
  // 7. 清空选中状态
  clearSelection();
}
```

## 七、待确认问题

### 7.1 章节间跳转
- Q: 章节之间如何跳转？
- A: 通过 `next` 字段指定下一章节，章节结束时自动跳转

### 7.2 跨章节引用
- Q: 节点能否引用其他章节的节点？
- A: 不可以，每个章节独立，跨章节通过章节跳转实现

### 7.3 起始节点
- Q: 每个章节是否必须有起始节点？
- A: 是的，每个章节需要指定一个起始节点

### 7.4 数据导入
- Q: 是否支持导入现有 YAML 文件？
- A: P1 功能，后续支持
