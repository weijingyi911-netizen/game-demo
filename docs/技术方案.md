### 1.2 技术选型及理由

#### 前端技术栈

| 技术 | 版本 | 选型理由 |
|-----|------|---------|
| Next.js | 14.x | React 全栈框架，支持 SSR/SSG，SEO 友好，开发效率高 |
| React | 18.x | 生态丰富，组件化开发，团队熟悉度高 |
| TypeScript | 5.x | 类型安全，提升代码质量和开发效率 |
| Ant Design | 5.x | 企业级 UI 组件库，中文文档完善，组件丰富 |
| ECharts | 5.x | 功能强大的可视化库，适合数据图表场景 |
| TailwindCSS | 3.x | 原子化 CSS，快速构建响应式界面 |
| Zustand | 4.x | 轻量级状态管理，API 简洁 |
| React Query | 5.x | 数据请求缓存管理，简化异步状态处理 |

#### 后端技术栈

| 技术 | 版本 | 选型理由 |
|-----|------|---------|
| Python | 3.11+ | AI 生态丰富，开发效率高，适合数据分析场景 |
| FastAPI | 0.100+ | 高性能异步框架，自动生成 API 文档，类型提示友好 |
| SQLAlchemy | 2.x | 成熟的 ORM，支持异步操作 |
| Pydantic | 2.x | 数据验证和序列化，与 FastAPI 深度集成 |
| LangChain | 0.1+ | LLM 应用开发框架，支持多模型切换 |
| Celery | 5.x | 分布式任务队列，处理异步任务 |
| Redis | 7.x | 高性能缓存，支持消息队列 |
| PostgreSQL | 15+ | 可靠的关系型数据库，支持 JSON 类型 |

#### AI 服务

| 服务 | 用途 | 选型理由 |
|-----|------|---------|
| OpenAI GPT-4 | 复杂推理分析 | 推理能力最强，适合深度分析场景 |
| 通义千问 Qwen | 中文场景分析 | 中文理解能力强，合规性好，成本低 |
| 文心一言 ERNIE | 备用模型 | 国产大模型，合规性好 |
| 向量数据库 (Milvus/Pinecone) | 知识库存储 | 存储商家数据向量化表示，支持语义检索 |

#### 基础设施

| 技术 | 用途 |
|-----|------|
| Docker | 容器化部署 |
| Docker Compose | 本地开发环境编排 |

#### 服务层设计

**诊断服务核心逻辑**

```python
class DiagnosisService:
    async def analyze_gmv_decline(
        self, 
        merchant_id: str,
        time_range: TimeRange
    ) -> DiagnosisReport:
        gmv_data = await self.data_service.get_gmv_metrics(merchant_id, time_range)
        
        factors = await self._decompose_gmv_factors(gmv_data)
        
        main_factor = self._identify_main_factor(factors)
        
        deep_analysis = await self._deep_analyze(merchant_id, main_factor, time_range)
        
        recommendations = await self.strategy_service.generate_recommendations(
            merchant_id, 
            diagnosis_result=deep_analysis
        )
        
        return DiagnosisReport(
            problem=DiagnosisProblem.GMV_DECLINE,
            summary=self._generate_summary(factors),
            factors=factors,
            deep_analysis=deep_analysis,
            recommendations=recommendations
        )
    
    async def _decompose_gmv_factors(self, gmv_data: GMVData) -> List[AnalysisFactor]:
        return [
            AnalysisFactor(
                name="流量",
                current_value=gmv_data.current.traffic,
                previous_value=gmv_data.previous.traffic,
                change_percent=self._calc_change_percent(
                    gmv_data.current.traffic, 
                    gmv_data.previous.traffic
                ),
                contribution=self._calc_contribution(
                    gmv_data.current.traffic / gmv_data.previous.traffic,
                    gmv_data.current.gmv / gmv_data.previous.gmv
                )
            ),
        ]
```

#### 核心表结构

**商家表 (merchants)**

```sql
CREATE TABLE merchants (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    name VARCHAR(255) NOT NULL,
    industry VARCHAR(100),
    platform VARCHAR(50),
    created_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP
);
```

**订单表 (orders)**

```sql
CREATE TABLE orders (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    merchant_id UUID NOT NULL REFERENCES merchants(id),
    order_no VARCHAR(100) NOT NULL,
    customer_id UUID,
    product_id UUID,
    quantity INTEGER NOT NULL,
    unit_price DECIMAL(10, 2) NOT NULL,
    total_amount DECIMAL(12, 2) NOT NULL,
    status VARCHAR(50) NOT NULL,
    channel VARCHAR(50),
    created_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP,
    paid_at TIMESTAMP WITH TIME ZONE,
    delivered_at TIMESTAMP WITH TIME ZONE
);

CREATE INDEX idx_orders_merchant_id ON orders(merchant_id);
CREATE INDEX idx_orders_created_at ON orders(created_at);
CREATE INDEX idx_orders_status ON orders(status);
```

**流量日志表 (traffic_logs)**

```sql
CREATE TABLE traffic_logs (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),

    date DATE NOT NULL,

    uv INTEGER NOT NULL DEFAULT 0,
    pv INTEGER NOT NULL DEFAULT 0,
    new_uv INTEGER NOT NULL DEFAULT 0,
    bounce_rate DECIMAL(5, 2),

);

CREATE INDEX idx_traffic_merchant_date ON traffic_logs(merchant_id, date);
CREATE INDEX idx_traffic_source ON traffic_logs(source);
```

### 2.4 AI 模块设计

#### LLM 统一调用层

```python
from abc import ABC, abstractmethod
from typing import Optional, Dict, Any
from pydantic import BaseModel

class LLMResponse(BaseModel):
    content: str
    model: str
    usage: Dict[str, int]
    finish_reason: str

class BaseLLMClient(ABC):
    @abstractmethod
    async def chat(
        self, 
        messages: list[Dict[str, str]],
        temperature: float = 0.7,
        max_tokens: int = 2000
    ) -> LLMResponse:
        pass
    
    @abstractmethod
    async def chat_with_context(
        self,
        query: str,
        context: str,
        system_prompt: Optional[str] = None
    ) -> LLMResponse:
        pass

class LLMFactory:
    _clients: Dict[str, BaseLLMClient] = {}
    
    @classmethod
    def get_client(cls, provider: str) -> BaseLLMClient:
        if provider not in cls._clients:
            if provider == "openai":
                cls._clients[provider] = OpenAIClient()
            elif provider == "qwen":
                cls._clients[provider] = QwenClient()
            elif provider == "ernie":
                cls._clients[provider] = ErnieClient()
        return cls._clients[provider]
```

#### LangChain 链设计

```python
from langchain.chains import LLMChain
from langchain.prompts import PromptTemplate
from langchain.output_parsers import PydanticOutputParser

class DiagnosisChain:
    def __init__(self, llm_client: BaseLLMClient):
        self.llm = llm_client
        self.parser = PydanticOutputParser(pydantic_object=DiagnosisReport)
        
        self.prompt = PromptTemplate(
            template=DIAGNOSIS_PROMPT,
            input_variables=[
                "industry", "platform", "time_range", 
                "data_summary", "user_question"
            ],
            partial_variables={
                "format_instructions": self.parser.get_format_instructions()
            }
        )
    
    async def run(
        self, 
        merchant: Merchant,
        time_range: TimeRange,
        question: str
    ) -> DiagnosisReport:
        data_summary = await self._prepare_data_summary(merchant.id, time_range)
        
        response = await self.llm.chat(
            messages=[
                {"role": "system", "content": self.prompt.format(
                    industry=merchant.industry,
                    platform=merchant.platform,
                    time_range=str(time_range),
                    data_summary=data_summary,
                    user_question=question
                )}
            ],
            temperature=0.3
        )
        
        return self.parser.parse(response.content)
```
